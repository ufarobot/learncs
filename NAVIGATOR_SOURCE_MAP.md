# Source Map для навигатора learncs.ru

Цель: не генерировать "авторские" материалы от лица Айрата, а сначала собрать карту реальных источников и только потом переносить отдельные тексты на сайт.

## Что было исправлено

- Черновые маршрутные страницы, написанные как общий ИИ-пересказ, удалены из исходников.
- `/navigator/` теперь ведет только на локальные страницы материалов внутри `/materials/`.
- Верхний слой навигатора теперь устроен как простой список локальных материалов без обложек, описаний, источников, времени чтения и внешних пост-ссылок.
- В самом навигаторе нельзя делать внешние ссылки на Habr/Telegram-посты и не нужно показывать источник как метку. Источник можно документировать внутри локальной страницы материала.
- Полные переносы материалов на сайт нужно делать отдельно: один источник, одна структура, одна редактура, потом публикация.

## Правило для следующих переносов

Нельзя писать локальную статью "по мотивам" нескольких источников и выдавать ее за авторскую позицию. Нужно:

1. Выбрать один исходный материал.
2. Выписать его структуру, тезисы, примеры и ссылки.
3. Согласовать, что именно переносим на сайт.
4. Сделать локальную версию без подмены голоса автора.
5. В навигаторе показывать только ссылку на локальную страницу материала.
6. В отдельной локальной статье источник можно документировать отдельно, но навигатор должен оставаться внутренним маршрутом чтения.
